# 📊 Прогресс-бар обработки поиска

## ✅ Что добавлено:

### Backend:

1. **Новые поля в Search entity:**
   - `processed_count` - количество обработанных резюме
   - `total_to_process` - общее количество резюме для обработки

2. **Обновление прогресса в Celery задачах:**
   - При обработке резюме прогресс обновляется после каждого резюме
   - При AI анализе прогресс обновляется после каждого анализа

3. **API возвращает прогресс:**
   - Все endpoints, возвращающие `SearchResponse`, теперь включают `processed_count` и `total_to_process`

### Frontend:

1. **Прогресс-бар:**
   - Отображается когда статус `processing` или `pending`
   - Показывает текущий прогресс: "X / Y"
   - Визуальный индикатор прогресса (LinearProgress)
   - Текстовое описание текущего этапа

2. **Автоматическое обновление:**
   - Прогресс обновляется каждые 2 секунды (как и статус)
   - Пользователь видит реальное время обработки

## 📋 Этапы обработки:

1. **Поиск резюме (process_search_task):**
   - `total_to_process` = количество найденных резюме
   - `processed_count` увеличивается после обработки каждого резюме

2. **AI анализ (analyze_top_resumes_task):**
   - `total_to_process` = количество резюме для глубокого анализа
   - `processed_count` увеличивается после анализа каждого резюме

## 🔄 Автоматическое обновление:

Фронтенд автоматически обновляет прогресс каждые 2 секунды, пока статус `processing` или `pending`:

```javascript
refetchInterval: (data) => {
  if (data?.status === 'processing' || data?.status === 'pending') {
    return 2000; // 2 секунды
  }
  return false;
}
```
