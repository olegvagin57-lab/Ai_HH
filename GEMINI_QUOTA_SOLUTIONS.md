# Решения проблемы с квотой Gemini API

## Текущая ситуация

Из Google Cloud Console видно:
- **Модель**: `gemini-2.5-flash`
- **RPD (Requests per day)**: 21/20 ❌ **ПРЕВЫШЕН**
- **RPM (Requests per minute)**: 4/5 ⚠️ близко к лимиту
- **TPM (Tokens per minute)**: 3.22K/250K ✅ в норме

## Проблема

Достигнут дневной лимит бесплатного плана: **20 запросов/день**

## Решения

### 1. Настроить биллинг (рекомендуется для production)

**Преимущества:**
- ✅ Увеличенные лимиты
- ✅ Больше запросов в день
- ✅ Стабильная работа

**Как сделать:**
1. В Google Cloud Console нажмите "Set up billing"
2. Добавьте способ оплаты
3. Лимиты автоматически увеличатся

**Новые лимиты с биллингом:**
- RPM: до 360 запросов/минуту
- RPD: до 1500 запросов/день
- TPM: до 1M токенов/минуту

### 2. Оптимизировать использование (бесплатно)

**Стратегии:**
- Кэшировать результаты анализа
- Анализировать только топ-кандидатов (не все резюме)
- Использовать batch-запросы где возможно
- Добавить задержки между запросами

**Текущие настройки в проекте:**
- `max_resumes_for_deep_analysis: 50` - можно уменьшить до 10-20
- Анализируются только топ резюме по preliminary_score

### 3. Использовать несколько API ключей (бесплатно)

**Стратегия:**
- Создать несколько Google аккаунтов
- Каждый аккаунт = 20 запросов/день
- Ротация ключей в Worker

**Недостатки:**
- ❌ Сложнее поддерживать
- ❌ Может нарушать ToS Google

### 4. Комбинировать с fallback режимом

**Стратегия:**
- Использовать Gemini для топ-10 резюме
- Остальные анализировать через fallback

**Реализация:**
Уже работает! Система автоматически переключается на fallback при ошибке 429.

### 5. Перейти на Hugging Face API (бесплатно)

**Преимущества:**
- ✅ 30,000 запросов/месяц бесплатно
- ✅ Не требует кредитную карту
- ✅ Хорошее качество

**Недостатки:**
- ❌ Нужна интеграция
- ❌ Может быть медленнее

## Рекомендации

### Для разработки/тестирования:
1. Использовать fallback режим (уже работает)
2. Ограничить количество анализируемых резюме
3. Ждать восстановления квоты (сбрасывается каждый день)

### Для production:
1. **Настроить биллинг** - лучший вариант
2. Оптимизировать использование (кэширование, батчинг)
3. Комбинировать с fallback для экономии

## Быстрое решение (прямо сейчас)

1. **Уменьшить количество анализируемых резюме:**
   ```python
   # В backend/app/config.py
   max_resumes_for_deep_analysis: int = 10  # вместо 50
   ```

2. **Использовать fallback режим:**
   - Уже работает автоматически
   - Базовое качество, но бесплатно

3. **Ждать до завтра:**
   - Квота сбрасывается каждый день
   - Завтра будет доступно 20 новых запросов
