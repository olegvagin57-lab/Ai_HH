# Настройка MCP (Model Context Protocol) серверов

## 📋 Обзор

MCP (Model Context Protocol) серверы предоставляют AI-ассистенту доступ к различным ресурсам и инструментам для более эффективной работы с проектом.

## 🚀 Установленные MCP серверы

### 1. **Filesystem** ✅
- **Назначение**: Доступ к файловой системе проекта
- **Использование**: Чтение, запись, поиск файлов в проекте
- **Конфигурация**: Автоматически использует корневую директорию проекта
- **Полезно для**: 
  - Анализа структуры проекта
  - Создания и редактирования файлов
  - Поиска по кодовой базе

### 2. **Sequential Thinking** ✅
- **Назначение**: Пошаговое мышление для сложных задач анализа
- **Использование**: Разбиение сложных задач на последовательные шаги
- **Конфигурация**: Не требует дополнительной настройки
- **Полезно для**: 
  - Глубокого анализа резюме по шагам
  - Структурированного подхода к сложным задачам
  - Логического разбора требований к вакансиям

### 3. **Context 7** ⚙️ (требует настройки)
- **Назначение**: Расширенное управление контекстом для детального анализа
- **Использование**: Работа с большими объемами данных и контекста
- **Требуется**: `CONTEXT7_API_KEY`
- **Как получить ключ**:
  1. Зарегистрируйтесь на Context7
  2. Получите API ключ
  3. Добавьте в `.env`: `CONTEXT7_API_KEY=your_key`
- **Полезно для**: 
  - Анализа больших резюме
  - Работы с расширенным контекстом вакансий
  - Детального сравнения кандидатов

### 4. **Playwright** ✅
- **Назначение**: Автоматизация браузера для веб-тестирования и парсинга
- **Использование**: Автоматизация браузера, тестирование, скрапинг
- **Конфигурация**: Не требует дополнительной настройки
- **Полезно для**: 
  - Тестирования фронтенда
  - Парсинга данных с HeadHunter
  - Автоматизации веб-задач
  - E2E тестирования

### 5. **WCGW (What Could Go Wrong)** ✅
- **Назначение**: Анализ рисков и выявление потенциальных проблем
- **Использование**: Обнаружение edge cases и потенциальных ошибок
- **Конфигурация**: Не требует дополнительной настройки
- **Полезно для**: 
  - Анализа рисков в резюме кандидатов
  - Выявления потенциальных проблем в коде
  - Проверки edge cases в логике анализа
  - Валидации данных

### 6. **Git** ✅
- **Назначение**: Работа с Git репозиторием
- **Использование**: Коммиты, ветки, история изменений
- **Конфигурация**: Автоматически использует корневую директорию проекта
- **Полезно для**: 
  - Управления версиями
  - Анализа истории изменений
  - Создания коммитов и веток

### 7. **MongoDB** ✅
- **Назначение**: Прямой доступ к базе данных MongoDB
- **Использование**: Запросы к коллекциям, анализ данных
- **Конфигурация**: 
  - `MONGODB_URI`: `mongodb://localhost:27017`
  - `MONGODB_DATABASE`: `hh_analyzer`
- **Полезно для**: 
  - Анализа резюме и поисков
  - Проверки данных кандидатов
  - Диагностики проблем с данными

### 8. **GitHub** ⚙️ (требует настройки)
- **Назначение**: Работа с GitHub API
- **Использование**: Создание issues, PR, управление репозиторием
- **Требуется**: `GITHUB_PERSONAL_ACCESS_TOKEN`
- **Как получить токен**:
  1. Перейдите в GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Создайте новый токен с правами: `repo`, `workflow`, `read:org`
  3. Добавьте токен в `.env` или переменные окружения
- **Полезно для**: 
  - Автоматизации работы с GitHub
  - Создания issues и PR
  - Управления репозиторием

### 9. **Brave Search** ⚙️ (требует настройки)
- **Назначение**: Поиск в интернете через Brave Search API
- **Использование**: Поиск информации, проверка данных
- **Требуется**: `BRAVE_API_KEY`
- **Как получить ключ**:
  1. Зарегистрируйтесь на https://brave.com/search/api/
  2. Получите API ключ
  3. Добавьте ключ в `.env` или переменные окружения
- **Полезно для**: 
  - Поиска информации о технологиях
  - Проверки актуальных данных
  - Исследований

### 10. **Puppeteer** ✅
- **Назначение**: Автоматизация браузера для веб-скрапинга
- **Использование**: Парсинг веб-страниц, автоматизация действий
- **Конфигурация**: Не требует дополнительной настройки
- **Полезно для**: 
  - Парсинга данных с HeadHunter
  - Тестирования фронтенда
  - Автоматизации веб-задач

### 11. **SQLite** ✅
- **Назначение**: Работа с SQLite базами данных
- **Использование**: Создание локальных БД для тестирования
- **Конфигурация**: Автоматически создает БД в `data/database.db`
- **Полезно для**: 
  - Локального тестирования
  - Кэширования данных
  - Аналитики

### 12. **PostgreSQL** ⚙️ (опционально)
- **Назначение**: Работа с PostgreSQL базами данных
- **Использование**: Если проект будет использовать PostgreSQL
- **Требуется**: `POSTGRES_CONNECTION_STRING`
- **Полезно для**: 
  - Миграции с MongoDB на PostgreSQL
  - Работы с реляционными данными

## 📝 Настройка

### Шаг 1: Установка конфигурации

1. Скопируйте шаблон конфигурации:
   ```powershell
   # Windows
   Copy-Item mcp.config.template.json "$env:APPDATA\Cursor\User\globalStorage\mcp.json"
   ```

   Или вручную:
   - Откройте файл `mcp.config.template.json` в корне проекта
   - Скопируйте его содержимое
   - В Cursor: Settings → MCP Servers → Add Configuration
   - Вставьте содержимое шаблона

2. Альтернативный способ (через настройки Cursor):
   - Откройте Cursor Settings (Ctrl+,)
   - Найдите раздел "MCP Servers"
   - Добавьте конфигурацию из `mcp.config.template.json`

### Шаг 2: Настройка переменных окружения

Для серверов, требующих API ключи, добавьте их в `.env` файл:

```env
# Context 7
CONTEXT7_API_KEY=your_context7_api_key_here

# GitHub
GITHUB_PERSONAL_ACCESS_TOKEN=your_github_token_here

# Brave Search
BRAVE_API_KEY=your_brave_api_key_here

# PostgreSQL (если используется)
POSTGRES_CONNECTION_STRING=postgresql://user:password@localhost:5432/dbname
```

### Шаг 3: Перезапуск Cursor

После настройки перезапустите Cursor IDE, чтобы MCP серверы загрузились.

## 🎯 Использование в проекте

### Для анализа резюме:
- **Sequential Thinking MCP**: Пошаговый анализ сложных резюме
- **Context 7 MCP**: Работа с большими объемами данных кандидатов
- **MongoDB MCP**: Прямой доступ к данным кандидатов
- **WCGW MCP**: Выявление рисков и проблем в резюме
- **Filesystem MCP**: Анализ кода обработки резюме
- **Git MCP**: Отслеживание изменений в логике анализа

### Для разработки:
- **Playwright MCP**: E2E тестирование фронтенда
- **GitHub MCP**: Автоматизация работы с репозиторием
- **Puppeteer MCP**: Тестирование и парсинг
- **Brave Search MCP**: Поиск документации и решений
- **WCGW MCP**: Анализ потенциальных проблем в коде

### Для отладки:
- **MongoDB MCP**: Проверка данных в БД
- **Filesystem MCP**: Анализ логов и конфигураций
- **SQLite MCP**: Локальное тестирование
- **WCGW MCP**: Выявление edge cases и проблем

## 🔧 Проверка работы

После перезапуска Cursor, MCP серверы должны автоматически подключиться. Вы можете проверить их работу, попросив AI-ассистента:

1. "Прочитай файл README.md" (Filesystem MCP)
2. "Покажи последние коммиты" (Git MCP)
3. "Сколько резюме в базе данных?" (MongoDB MCP)

## ⚠️ Важные замечания

1. **Безопасность**: Не коммитьте API ключи в Git. Используйте `.env` файл и добавьте его в `.gitignore`.

2. **Производительность**: MCP серверы работают асинхронно, но некоторые операции могут занять время.

3. **Ограничения**: 
   - GitHub MCP требует токен с соответствующими правами
   - Brave Search имеет лимиты на количество запросов
   - Puppeteer требует установленного Chrome/Chromium
