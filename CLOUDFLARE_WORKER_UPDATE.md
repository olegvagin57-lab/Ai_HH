# Обновление Cloudflare Worker

## Проблема
Текущий Worker просто проксирует запросы к Gemini API, но не обрабатывает наши кастомные endpoints `/extract_concepts` и `/analyze_resume`.

## Решение
Обновленный Worker теперь:
1. ✅ Обрабатывает `/extract_concepts` - извлекает концепты из запроса через Gemini
2. ✅ Обрабатывает `/analyze_resume` - анализирует резюме через Gemini
3. ✅ Сохраняет обратную совместимость - проксирует другие запросы напрямую

## Инструкция по обновлению

### Шаг 1: Откройте Cloudflare Dashboard
1. Перейдите на https://dash.cloudflare.com
2. Выберите ваш аккаунт
3. Перейдите в **Workers & Pages**
4. Найдите Worker `proud-water-5293` (или создайте новый)

### Шаг 2: Обновите код Worker
1. Откройте Worker для редактирования
2. Скопируйте содержимое файла `cloudflare_worker.js` из корня проекта
3. Вставьте его в редактор кода Worker
4. **ВАЖНО**: Убедитесь, что API ключ Gemini правильный (он уже в коде)

### Шаг 3: Разверните Worker
1. Нажмите **Save and Deploy**
2. Дождитесь успешного развертывания

### Шаг 4: Протестируйте
Запустите тест:
```bash
cd backend
python scripts/test_parser_gemini.py
```

Или проверьте Worker напрямую:
```bash
python scripts/check_gemini_status.py
```

## Что изменилось

### Старый Worker
- Просто проксировал запросы к Gemini API
- Не обрабатывал `/extract_concepts` и `/analyze_resume`
- Возвращал 404 для наших endpoints

### Новый Worker
- ✅ Обрабатывает `/extract_concepts`:
  - Принимает `{"query": "текст запроса"}`
  - Формирует prompt для Gemini
  - Парсит ответ и возвращает `{"concepts": [["концепт1", "синоним1"], ...]}`

- ✅ Обрабатывает `/analyze_resume`:
  - Принимает `{"resume_text": "...", "concepts": [...], "vacancy_requirements": {...}}`
  - Формирует детальный prompt для анализа
  - Парсит ответ и возвращает полный анализ с оценками

- ✅ Сохраняет обратную совместимость:
  - Другие запросы проксируются напрямую к Gemini API

## Проверка работы

После обновления Worker должен:
1. ✅ Возвращать 200 для POST `/extract_concepts`
2. ✅ Возвращать 200 для POST `/analyze_resume`
3. ✅ Возвращать концепты в правильном формате
4. ✅ Возвращать анализ резюме со всеми полями

## Если что-то не работает

1. **Проверьте логи Worker** в Cloudflare Dashboard
2. **Проверьте API ключ Gemini** - возможно, закончилась квота
3. **Проверьте формат запросов** - используйте тестовые скрипты
4. **Проверьте CORS** - Worker должен возвращать правильные заголовки

## Альтернатива

Если не хотите обновлять Worker, можно:
1. Использовать fallback режим (уже работает)
2. Настроить прямое подключение к Gemini API (нужен API ключ)
