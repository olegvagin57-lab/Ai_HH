# Руководство по ручному тестированию

## Дата создания
2026-01-17

## Подготовка тестовых данных

### Запуск скрипта подготовки данных

```powershell
cd backend
docker exec hh_analyzer_backend python scripts/prepare_test_data.py
```

Скрипт создаст:
- **3 тестовых пользователя** (admin, hr, manager)
- **3 вакансии** (Python Developer, React Developer, DevOps Engineer)
- **3 поиска резюме** (разные статусы)
- **3 резюме** (для тестирования)
- **2 кандидата** (связанные с резюме)
- **3 уведомления** (разные типы)

### Тестовые учетные записи

| Email | Password | Role | Описание |
|-------|----------|------|----------|
| admin@test.com | Admin123! | admin | Полный доступ |
| hr@test.com | Hr123456! | hr_specialist | HR специалист |
| manager@test.com | Manager123! | hr_manager | HR менеджер |

## Сценарии ручного тестирования

### Сценарий 1: Регистрация и первый вход

**Цель**: Проверить процесс регистрации и первого входа

**Шаги**:
1. Открыть frontend: http://localhost:3000
2. Перейти на страницу регистрации
3. Заполнить форму:
   - Email: `newuser@test.com`
   - Username: `newuser`
   - Password: `NewUser123!`
   - Full Name: `New User`
4. Нажать "Зарегистрироваться"
5. Проверить успешную регистрацию
6. Войти с созданными учетными данными
7. Проверить доступ к основному функционалу

**Ожидаемый результат**:
- ✅ Регистрация успешна
- ✅ Пользователь автоматически залогинен
- ✅ Доступ к основным функциям

---

### Сценарий 2: Создание поиска резюме

**Цель**: Проверить создание и запуск поиска резюме

**Шаги**:
1. Войти как `hr@test.com / Hr123456!`
2. Перейти в раздел "Поиск резюме"
3. Нажать "Создать новый поиск"
4. Заполнить форму:
   - Query: "Python разработчик"
   - Area: "Москва"
   - Specialization: "Разработка"
5. Нажать "Создать поиск"
6. Проверить статус поиска (должен быть "pending" или "processing")
7. Дождаться завершения поиска (статус "completed")
8. Проверить найденные резюме

**Ожидаемый результат**:
- ✅ Поиск создан успешно
- ✅ Поиск обрабатывается
- ✅ Найдены резюме
- ✅ Резюме отображаются в списке

- [ ] Только авторизованные пользователи имеют доступ
- [ ] Роли и права работают корректно
- [ ] Данные защищены
- [ ] Нет утечек информации

## Заметки для тестировщика

1. **Используйте разные браузеры**: Chrome, Firefox, Edge
2. **Проверяйте на разных устройствах**: Desktop, Tablet, Mobile
3. **Тестируйте граничные случаи**: пустые данные, очень длинные строки, специальные символы
4. **Проверяйте параллельные действия**: несколько пользователей одновременно
5. **Документируйте найденные баги**: скриншоты, шаги воспроизведения, ожидаемое/фактическое поведение
